# แก้ไขปัญหาเลขที่ใบกำกับภาษี

## ปัญหาที่แก้ไข

### 1. ข้อผิดพลาด "Unexpected token '<', ... is not valid JSON"
**สาเหตุ:** API ส่งค่า HTML หรือ PHP error กลับมาแทน JSON

**การแก้ไข:**
- เพิ่ม `error_reporting()` และ `ini_set('display_errors', 0)` ใน API
- ตรวจสอบ Content-Type ก่อนแปลง response เป็น JSON
- เพิ่ม error handling ที่ดีขึ้นใน JavaScript

### 2. เลขที่ใบกำกับภาษีซ้ำกันระหว่างประเภทเอกสาร
**สาเหตุ:** ต้องการให้แต่ละประเภทเอกสารมีเลขที่แยกกัน

**การแก้ไข:**
- แก้ไข API `get_next_invoice_number.php` ให้กรอง `doc_type`
- แก้ไข database constraint จาก `UNIQUE(inv_no)` เป็น `UNIQUE(inv_no, doc_type)`
- อัปเดต UI ให้สร้างเลขใหม่เมื่อเปลี่ยนประเภทเอกสาร

## วิธีใช้งาน

### 1. รัน Migration Script
เปิด browser ไปที่:
```
http://localhost/fix_invoice_unique_constraint.php
```

หรือรันผ่าน command line:
```bash
php fix_invoice_unique_constraint.php
```

Script นี้จะ:
- ตรวจสอบ constraint ปัจจุบัน
- ลบ UNIQUE constraint เก่าที่มีแค่ `inv_no`
- สร้าง UNIQUE constraint ใหม่ `(inv_no, doc_type)`

### 2. ทดสอบการทำงาน

1. เปิดหน้าสร้างใบกำกับภาษี
2. เลือกประเภทเอกสาร "ใบกำกับภาษี"
3. คลิกปุ่ม 🔄 (Refresh) เพื่อสร้างเลขที่ เช่น `202604-001`
4. เปลี่ยนเป็น "ใบเสนอราคา"
5. ระบบจะสร้างเลขใหม่ เช่น `202604-001` (ซ้ำได้เพราะคนละประเภท)
6. บันทึกเอกสารทั้งสองได้สำเร็จ

## ตัวอย่างการใช้งาน

### กรณีที่ 1: เอกสารประเภทเดียวกัน
```
ใบกำกับภาษี: 202604-001 ✓
ใบกำกับภาษี: 202604-002 ✓
ใบกำกับภาษี: 202604-003 ✓
```

### กรณีที่ 2: เอกสารคนละประเภท (เลขซ้ำได้)
```
ใบกำกับภาษี: 202604-001 ✓
ใบเสนอราคา: 202604-001 ✓ (ซ้ำได้)
ใบแจ้งหนี้:   202604-001 ✓ (ซ้ำได้)
ใบสำคัญจ่าย: 202604-001 ✓ (ซ้ำได้)
```

### กรณีที่ 3: เดือนใหม่ รีเซ็ต
```
เมษายน 2026:
  - ใบกำกับภาษี: 202604-001, 202604-002
  - ใบเสนอราคา: 202604-001, 202604-002

พฤษภาคม 2026:
  - ใบกำกับภาษี: 202605-001 (เริ่มใหม่)
  - ใบเสนอราคา: 202605-001 (เริ่มใหม่)
```

## คุณสมบัติ

### 1. เลขอัตโนมัติ
- รูปแบบ: `YYYYMM-NNN` (เช่น 202604-001)
- ดึงเลขล่าสุดจากฐานข้อมูล
- แยกตามประเภทเอกสาร
- เริ่มใหม่ทุกเดือน

### 2. ปุ่ม Refresh
- คลิกเพื่อสร้างเลขใหม่
- แสดงแอนิเมชันขณะโหลด
- แสดง toast notification เมื่อสำเร็จ

### 3. แก้ไขเอง
- สามารถพิมพ์เลขเองได้
- ไม่บังคับต้องใช้เลขอัตโนมัติ

### 4. เปลี่ยนประเภทเอกสาร
- เมื่อเปลี่ยนประเภท จะสร้างเลขใหม่อัตโนมัติ
- แต่ละประเภทมี sequence แยกกัน

## ไฟล์ที่แก้ไข

1. **api/get_next_invoice_number.php** (ใหม่)
   - API สร้างเลขที่อัตโนมัติ
   - รองรับการแยก doc_type

2. **reports/tax_invoice.php**
   - เพิ่มปุ่ม refresh
   - เพิ่มฟังก์ชันสร้างเลขอัตโนมัติ
   - ปรับ UI

3. **fix_invoice_unique_constraint.php** (ใหม่)
   - Migration script แก้ไข database constraint

## หมายเหตุ

- ตรวจสอบให้แน่ใจว่ารัน migration script ก่อนใช้งาน
- ถ้ามีข้อมูลซ้ำอยู่แล้ว ต้องแก้ไขก่อนรัน migration
- สามารถ rollback โดยสร้าง UNIQUE constraint แค่ inv_no อีกครั้ง
