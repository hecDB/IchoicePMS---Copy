# IchoicePMS — ขอบเขตและความสามารถของระบบ

**IchoicePMS** คือระบบบริหารจัดการคลังสินค้าและการซื้อขาย (Product Management System)  
สำหรับธุรกิจที่ต้องการติดตามสินค้า คำสั่งซื้อ การขาย และเอกสารทางบัญชีในที่เดียว

---

## ภาพรวมระบบ

| หัวข้อ | รายละเอียด |
|--------|-----------|
| ภาษา | PHP + MySQL |
| ภาษาที่แสดงผล | ภาษาไทย |
| ผู้ใช้งาน | Admin, Manager, User (แบ่งสิทธิ์) |
| รองรับ | Multi-currency (หลายสกุลเงิน) |
| Platform ขาย | Shopee, Lazada (ผ่าน tag pattern) |

---

## เมนูและความสามารถ

---

### 1. แดชบอร์ด (Dashboard)

ภาพรวมสถานะระบบในหน้าเดียว

**ทำได้:**
- ดูจำนวน PO ที่รอรับสินค้า (pending/partial)
- ดูจำนวนผู้ใช้ที่รออนุมัติ (เฉพาะ admin)
- บังคับเปลี่ยนรหัสผ่านครั้งแรก (first-login)

---

### 2. จัดการคำสั่งซื้อ (Purchase Orders)

บริหารใบสั่งซื้อตั้งแต่สร้างจนถึงรับสินค้า

#### รายการใบสั่งซื้อ
- ดูรายการ PO ทั้งหมด พร้อมชื่อ supplier, วันที่, ยอดเงิน
- กรองสถานะ: pending / partial / complete
- ดูรายละเอียด PO แต่ละรายการ
- แก้ไขและลบ PO

#### สร้างใบสั่งซื้อใหม่
- สร้าง PO สำหรับสินค้าที่มีในระบบอยู่แล้ว
- เลือก supplier, ระบุราคา, จำนวน, สกุลเงิน
- ระบบตรวจสอบเลข PO ซ้ำอัตโนมัติ
- รองรับหลายรายการสินค้าต่อ 1 PO

#### ซื้อสินค้าใหม่ *(admin/manager เท่านั้น)*
- สร้าง PO พร้อมสินค้าใหม่ที่ยังไม่มีในระบบ (temp product)
- เลือกหมวดหมู่และสกุลเงิน
- สินค้าใหม่รอการอนุมัติก่อนเข้าระบบถาวร

#### รับเข้าสินค้า
- แสดง PO ที่รอรับพร้อม progress bar รายการ
- บันทึกการรับสินค้าพร้อม: จำนวน, ราคาต้นทุน, ตำแหน่งจัดเก็บ (row-bin-shelf), วันหมดอายุ
- รองรับรับสินค้าบางส่วน (partial receive)
- บันทึกสินค้าชำรุดขณะรับ

#### รับสินค้าด่วน (Scan)
- สแกน barcode เพื่อรับสินค้าแบบเร็ว
- เหมาะสำหรับรับสินค้าที่ทราบข้อมูลแล้ว

---

### 3. อัปโหลดสินค้า (Import/Upload)

นำเข้าข้อมูลสินค้าจากไฟล์ Excel จำนวนมากในครั้งเดียว

**ทำได้:**
- อัปโหลดไฟล์ `.xlsx` เพื่อนำเข้าสินค้าจำนวนมาก
- mapping หมวดหมู่สินค้าอัตโนมัติ
- ตรวจสอบข้อมูลก่อน import (validation)
- ดาวน์โหลด Excel template สำหรับกรอกข้อมูล
- สแกน barcode เพื่อระบุสินค้าระหว่าง import

---

### 4. จัดการสต็อก (Stock Management)

ติดตามความเคลื่อนไหวและสถานะสินค้าในคลัง

#### ความเคลื่อนไหวสินค้า
- แสดงสินค้าที่รับเข้าทั้งหมด พร้อมตำแหน่งจัดเก็บ, สต็อกคงเหลือ, ราคา
- แสดง badge จำนวน PO ที่รอรับ

#### สินค้าซื้อใหม่
- รายการสินค้าที่เพิ่งซื้อเข้ามา แยกมุมมองจาก stock ทั่วไป

#### สินค้าใกล้หมด
- แสดงสินค้าที่มีสต็อก ≤ 5 ชิ้น
- แบ่งระดับความเร่งด่วน: critical / low / normal

#### จัดการสินค้า
- รายการสินค้าทั้งหมดแบบครบถ้วน
- ดูหมวดหมู่, ตำแหน่งจัดเก็บ, จำนวนสต็อก
- ติดตาม activity ของสินค้าแต่ละชิ้น

#### สินค้าใกล้หมดอายุ
- แสดงสินค้าที่จะหมดอายุภายใน 300 วัน
- แสดงราคาต้นทุน/ราคาขาย และจำนวนวันที่เหลือ
- แบ่งระดับ: critical / warning / caution
- รับทราบการแจ้งเตือน (acknowledge)

#### สินค้าชำรุดบางส่วน
- ฟอร์มตรวจสอบสินค้าชำรุดหรือบกพร่องที่พบระหว่างรับ
- บันทึกผลตรวจสอบและจัดหมวดหมู่การตีกลับ
- ติดตามสถานะสินค้าที่ชำรุดแต่ยังขายได้บางส่วน

---

### 5. สินค้านำออก (Taken Out / Missing Products)

บันทึกสินค้าที่ออกจากคลังโดยไม่ใช่การขายปกติ

**ทำได้:**
- บันทึกสินค้าสูญหาย, เสียหาย, หรือนำออกกรณีอื่นๆ
- สร้าง negative inventory transaction อัตโนมัติ
- ดูสถิติสินค้าที่หายไป: จำนวนรายการ, มูลค่า
- แก้ไขและลบรายการที่บันทึกผิด
- ค้นหาและกรองรายการ

---

### 6. ขายสินค้า (Sales / Issue Products)

จัดการการขายและออกสินค้าสำหรับ e-commerce

#### ยิงสินค้าออก (ขาย)
- สแกน barcode หรือค้นหาสินค้าเพื่อขาย
- ตรวจสอบ tag pattern (Shopee/Lazada) ก่อนยืนยัน
- บันทึกการขายพร้อมเลขแท็ก platform
- ตัดสต็อกอัตโนมัติเมื่อขาย

#### รายการขาย
- แสดงรายการขายทั้งหมด
- ดูเลขแท็ก, วันที่, จำนวนสินค้าต่อรายการ
- ติดตามประวัติการขายย้อนหลัง

#### จัดการเลขแท็ก
- สร้างและแก้ไข tag patterns สำหรับ Shopee และ Lazada
- กำหนดรูปแบบเลขแท็ก (format/pattern)
- ดูสถิติ patterns ที่ใช้งาน

---

### 7. รายงาน (Reports)

วิเคราะห์ข้อมูลและส่งออกรายงาน

#### รายงานประเภทสินค้า
- แสดงสินค้าแยกตามหมวดหมู่
- กรองด้วย date range (สูงสุด 90 วัน)
- ดูยอดรวมต่อ category

#### รายงานคำสั่งซื้อ
- วิเคราะห์ purchase orders ตามช่วงเวลา
- กรองวันที่, supplier, สถานะ
- ดูมูลค่าคำสั่งซื้อรวม

#### รายงานสินค้า
- รายงาน inventory สินค้าโดยละเอียด
- ดูสต็อกคงเหลือ, ราคา, ตำแหน่ง
- แสดงรูปสินค้า (image resolution helper)

---

### 8. ใบกำกับภาษี (Tax Invoices)

สร้างและจัดการเอกสารใบกำกับภาษีมูลค่าเพิ่ม

#### สร้างใบกำกับภาษี
- กรอกข้อมูลผู้ซื้อ/ผู้ขาย
- ระบุรายการสินค้า จำนวน ราคา
- คำนวณภาษีมูลค่าเพิ่ม (VAT) อัตโนมัติ
- สร้างเลขที่ใบกำกับภาษีอัตโนมัติ

#### ใบกำกับภาษีที่บันทึก
- แสดงรายการใบกำกับภาษีทั้งหมด
- กรองด้วย date range
- ส่งออกเป็น Excel
- ดูรายละเอียดและพิมพ์ใบกำกับ

---

### 9. สินค้าตีกลับ (Returns)

บริหารจัดการสินค้าที่ถูกส่งคืน

#### บันทึกสินค้าตีกลับ
- บันทึกสินค้าที่ลูกค้าตีกลับ/ส่งคืน
- ระบุเหตุผลการตีกลับ
- เชื่อมโยงกับรายการขายต้นทาง
- อัปเดตสต็อกคืนอัตโนมัติ

#### จัดการสินค้าตีกลับ
- dashboard ภาพรวมสินค้าตีกลับทั้งหมด
- ติดตามสถานะ: รอตรวจสอบ / ผ่านแล้ว / ยกเลิก
- ดูประวัติและเหตุผลการตีกลับ
- จัดการ damaged goods ที่ตีกลับ

---

### 10. ผู้ดูแลระบบ / Admin *(เฉพาะ role: admin)*

#### จัดการผู้ใช้งาน
- ดูรายการผู้ใช้ทั้งหมด (อนุมัติแล้ว / รออนุมัติ)
- อนุมัติ / ปฏิเสธ ผู้ใช้ใหม่
- แก้ไขข้อมูลผู้ใช้ (ชื่อ, email, แผนก, role)
- ลบผู้ใช้ออกจากระบบ
- รีเซ็ตรหัสผ่านและสร้าง temporary password
- แสดง badge จำนวนรออนุมัติบน sidebar

#### จัดการสกุลเงิน
- เพิ่ม/แก้ไขสกุลเงินที่ใช้ใน PO
- กำหนดอัตราแลกเปลี่ยน
- กำหนด base currency (สกุลเงินหลัก)

---

## ระบบสนับสนุน (ไม่ใช่เมนู)

### Authentication & Security
- ระบบ login/register พร้อม workflow อนุมัติผู้ใช้ใหม่
- บังคับเปลี่ยนรหัสผ่านครั้งแรก (ต้องมีตัวพิมพ์ใหญ่, เล็ก, ตัวเลข, อักขระพิเศษ ≥ 8 ตัว)
- ระบบลืมรหัสผ่าน (forgot password request)
- Session-based authentication

### Role-Based Access Control
| Role | สิทธิ์ |
|------|--------|
| admin | เข้าถึงทุกเมนู รวมถึง Admin panel |
| manager | เข้าถึงได้รวมถึงซื้อสินค้าใหม่ |
| user | เข้าถึงเมนูทั่วไป ไม่มี Admin panel |

### Multi-Currency Support
- รองรับหลายสกุลเงินใน purchase order
- คำนวณอัตราแลกเปลี่ยนอัตโนมัติ
- แสดงราคาในสกุลเงินที่กำหนด

### ระบบตำแหน่งจัดเก็บ (Location)
- กำหนดตำแหน่งแบบ row-bin-shelf (แถว-ช่อง-ชั้น)
- ค้นหาสินค้าจากตำแหน่งได้
- อัปเดตตำแหน่งเมื่อรับสินค้า

### การแจ้งเตือน (Notifications)
- badge บน sidebar แจ้งจำนวน PO รอรับสินค้า
- badge แจ้งจำนวนผู้ใช้รออนุมัติ (admin)
- แจ้งเตือนสินค้าใกล้หมดอายุ

---

## สิ่งที่ระบบยังไม่รองรับ

- ระบบ POS (Point of Sale) แบบ real-time หน้าร้าน
- การส่งอีเมลแจ้งเตือนอัตโนมัติ
- Mobile app (รองรับเฉพาะ web browser)
- การ sync กับระบบ Shopee/Lazada โดยตรง (ใช้ tag pattern เท่านั้น)
- ระบบบัญชี (GL, AP, AR) แบบเต็มรูปแบบ
