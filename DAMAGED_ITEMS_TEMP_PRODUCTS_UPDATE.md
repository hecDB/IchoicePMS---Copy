# 🔄 อัปเดตระบบตรวจสอบสินค้าชำรุด - บันทึก temp_products

## 📋 รายละเอียดการเปลี่ยนแปลง

### ✨ ฟีเจอร์ใหม่
สินค้าชำรุดที่ **ขายได้** จะถูกบันทึกลงใน **ตาราง `temp_products`** แทนการสร้าง PO ใหม่

### 🎯 วัตถุประสงค์
- **ลดความยุ่งความปลอดภัย**: ไม่สร้าง PO หลากหลายสำหรับสินค้าชำรุด
- **กระบวนการอนุมัติชัดเจน**: สินค้าผ่านการทำสัญญาก่อนเข้าสต๊อก
- **ติดตาม SKU ใหม่**: ผู้ดูแลต้องอนุมัติและสร้างสินค้าใน `products` ตาราง

---

## 🔧 การทำงาน

### ขั้นตอน 1: ตรวจสอบสินค้าชำรุด
1. เปิดหน้า **ตรวจสอบสินค้าชำรุดบางส่วน**
2. เลือกสินค้าจากรายการทางด้านซ้าย
3. กรอกข้อมูล:
   - **SKU เดิม**: ระบบแสดงอัตโนมัติ
   - **SKU ใหม่**: ระบบสร้าง "ตำหนิ-{SKU เดิม}"
   - **จำนวนที่จะนำกลับ**: ระบุจำนวนสินค้า
   - **ราคาทุน/ขาย**: ตั้งค่าราคาใหม่อย่างไรก็ได้
   - **หมายเหตุการตรวจสอบ**: บันทึกสภาพสินค้าหรือรหัสตำหนิ

### ขั้นตอน 2: เลือกสถานะสินค้า
- **✅ ขายได้**: บันทึกเข้า `temp_products` + ปล่อยให้อนุมัติ
- **❌ ทิ้ง/ไม่ได้**: ปิดงานเท่านั้น (ไม่บันทึก)

### ขั้นตอน 3: ยืนยันและบันทึก
- คลิกปุ่ม **"ยืนยันสินค้ามีตำหนิและบันทึก temp_products"**
- ระบบจะบันทึกข้อมูลลงใน `temp_products` ตาราง
- สถานะจะเปลี่ยนเป็น "ตรวจสอบแล้ว"

---

## 📊 ข้อมูลที่บันทึกลง temp_products

```
┌─ temp_products
│
├─ product_name: ชื่อสินค้าเดิม
├─ product_category: หมวดหมู่สินค้า
├─ product_image: รูปภาพสินค้า
├─ unit: หน่วยนับ
├─ provisional_sku: SKU ใหม่ (ตำหนิ-{SKU})
├─ provisional_barcode: barcode เดิม
├─ remark: หมายเหตุ + รหัสตีกลับ
├─ status: "pending_approval"
├─ po_id: PO ต้นทาง (ถ้ามี)
├─ created_by: ผู้บันทึก
└─ created_at: วันเวลาบันทึก
```

---

## 👤 ขั้นตอนสำหรับผู้ดูแลระบบ (Admin)

### 1️⃣ ตรวจสอบ temp_products
1. ไปยังหน้า **"สินค้าใหม่ที่รอการอนุมัติ"**
2. ค้นหาและตรวจสอบสินค้าชำรุด (เรียบร้อยแล้วจาก remark)

### 2️⃣ อนุมัติและสร้างสินค้า
1. ตรวจสอบ SKU ใหม่ว่าไม่ซ้ำ
2. เพิ่มข้อมูล:
   - ราคาทุน/ขายสุดท้าย
   - หมวดหมู่
3. คลิก **"อนุมัติ"** เพื่อสร้างสินค้าใหม่ใน `products`

### 3️⃣ ตรวจสอบการเคลื่อนไหวสินค้า
- ตรวจสอบ `product_activity` สำหรับบันทึกของสินค้าชำรุด

---

## 🔄 เปรียบเทียบกระบวนการเดิม vs ใหม่

### ❌ กระบวนการเดิม
```
สินค้าชำรุด → สร้าง PO ใหม่ → สร้าง receive_items → เข้าสต๊อก
```

### ✅ กระบวนการใหม่
```
สินค้าชำรุด (ขายได้) → temp_products → รอการอนุมัติ SKU ใหม่ → สร้างสินค้า → เข้าสต๊อก
สินค้าชำรุด (ทิ้ง) → ปิดงาน (ไม่บันทึก)
```

## ⚠️ หมายเหตุสำคัญ

1. **SKU ใหม่ต้องไม่ซ้ำ**: ไม่สามารถมี SKU ที่เหมือนกันได้
2. **การตรวจสอบต้องเสร็จ**: ต้องกดปุ่มยืนยันเพื่อให้บันทึก
3. **ไม่สามารถยกเลิกได้**: เมื่อตรวจสอบแล้ว จะปิดการแก้ไข
4. **ข้อมูลต้องสมบูรณ์**: ต้องมีจำนวนและสถานะ

## 🆘 การแก้ปัญหาทั่วไป

### ❓ ทำไมไม่สร้าง PO หลากหลาย?
→ เพื่อให้สินค้าชำรุดผ่านการอนุมัติ SKU ใหม่ก่อน

### ❓ สินค้า "ทิ้ง" ไปไหน?
→ ไม่บันทึก - ปิดงานเท่านั้น (จัดประเมินว่าไม่ใช้ได้)

### ❓ ต้องทำอะไรหลังตรวจสอบเสร็จ?
→ สำหรับ ขายได้: Admin อนุมัติใน temp_products
สำหรับ ทิ้ง: เสร็จแล้ว ไม่ต้องทำอะไร
