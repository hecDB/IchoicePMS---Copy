🔌 API ENDPOINTS

  get_reasons
    └─ GET /api/returned_items_api.php?action=get_reasons
       └─ ดึงเหตุผลการตีกลับ

  search_po
    └─ GET /api/returned_items_api.php?action=search_po&keyword=PO-XXXX
       └─ ค้นหา PO

  get_po_items
    └─ GET /api/returned_items_api.php?action=get_po_items&po_id=1
       └─ ดึงรายการสินค้าใน PO

  create_return
    └─ POST /api/returned_items_api.php
       └─ สร้างบันทึกตีกลับ

  get_returns
    └─ GET /api/returned_items_api.php?action=get_returns
       └─ ดึงรายการตีกลับ

  get_return
    └─ GET /api/returned_items_api.php?action=get_return&return_id=1
       └─ ดึงรายละเอียดการตีกลับ

  approve_return
    └─ POST /api/returned_items_api.php
       └─ อนุมัติการตีกลับ

  reject_return
    └─ POST /api/returned_items_api.php
       └─ ปฏิเสธการตีกลับ

✨ KEY FEATURES

  ✓ ค้นหา PO ตามเลขที่ PO หรือเลขที่ออกสินค้า
  ✓ ตรวจสอบสินค้าที่สามารถตีกลับได้
  ✓ บันทึกจำนวนและเหตุผล
  ✓ กรอกหมายเหตุเพิ่มเติม
  ✓ เหตุผลอัตโนมัติแสดงว่า returnable หรือ non-returnable
  ✓ ดูแดชบอร์ด 4 stat cards
  ✓ ตาราง รายการตีกลับ
  ✓ ค้นหาและกรองตามสถานะ/ประเภท
  ✓ ดูรายละเอียดการตีกลับ
  ✓ อนุมัติการตีกลับ
  ✓ ปฏิเสธพร้อมเหตุผล
  ✓ Auto-refresh ทุก 30 วินาที

💡 TIPS & BEST PRACTICES

  ✓ ใช้เลขที่ PO ที่ถูกต้องเพื่อค้นหา
  ✓ บันทึกหมายเหตุโดยละเอียดเพื่อติดตามง่าย
  ✓ อนุมัติการตีกลับทันทีเพื่อให้ทันคืนสต็อก
  ✓ ตรวจสอบแดชบอร์ดเป็นประจำ
  ✓ ปฏิเสธพร้อมเหตุผลชัดเจน

🐛 TROUBLESHOOTING

  ❓ ไม่สามารถค้นหา PO
     └─ ตรวจสอบว่าเลขที่ PO ถูกต้อง
     └─ ตรวจสอบว่า PO มีสินค้าที่ได้รับแล้ว

  ❓ ไม่สามารถสร้างตารางฐานข้อมูล
     └─ ตรวจสอบการเชื่อมต่อ MySQL
     └─ ตรวจสอบสิทธิ์ของผู้ใช้
     └─ ตรวจสอบชื่อฐานข้อมูล

  ❓ เกิด API error
     └─ ตรวจสอบ Browser Console
     └─ ตรวจสอบ Network Tab
     └─ ตรวจสอบ Server logs
